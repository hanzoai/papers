% ABSTRACT
% ============================================================================
\begin{abstract}
We present Hanzo Crypto Payment Rails, an extension of the Hanzo Commerce
platform that enables merchants to accept cryptocurrency payments (BTC,
ETH, USDC, USDT) with instant settlement on Lux Network. Built on the
foundation of Hanzo's checkout.js library (in production since 2015),
the crypto payment extension uses MPC threshold signing for merchant
wallet management, multi-chain transaction monitoring (EVM, Bitcoin,
Solana), and automatic fiat off-ramps. We describe the architecture:
payment channel creation, multi-chain address derivation via a single MPC
key ceremony, real-time exchange rate locking, settlement finality
guarantees, and integration with the Hanzo Commerce API. Production
benchmarks from merchant deployments show median payment confirmation in
1.2 seconds on Lux, 12 minutes on Bitcoin, and 0.4 seconds on Solana,
with settlement to merchant fiat accounts within 24 hours.
\end{abstract}

% ============================================================================
\section{Introduction}
\label{sec:intro}
% ============================================================================

Cryptocurrency payment adoption by merchants remains limited despite a
\$2.5 trillion total market capitalization~\cite{coinmarket2024}. The
primary barriers are: (i)~exchange rate volatility between payment
initiation and settlement, (ii)~slow confirmation times on Bitcoin and
Ethereum mainnet, (iii)~key management complexity for merchant wallets,
and (iv)~fragmented integration across chains.

Hanzo Commerce has processed merchant transactions since 2015 through
its checkout.js JavaScript SDK---a drop-in payment widget supporting
credit cards, Apple Pay, and Google Pay. The crypto payment extension
adds cryptocurrency acceptance as a first-class payment method within
the same SDK, same dashboard, and same settlement pipeline.

% ============================================================================
\section{Checkout.js Heritage}
\label{sec:checkout}
% ============================================================================

Hanzo checkout.js has been in continuous production since 2015, powering
payment flows for thousands of merchants. Key properties:

\begin{itemize}[leftmargin=1.4em]
    \item \textbf{Drop-in integration}: Single \texttt{<script>} tag.
    No server-side code required.
    \item \textbf{PCI DSS compliant}: Card data never touches merchant
    servers. Tokenized via Hanzo's PCI Level 1 vault.
    \item \textbf{Multi-currency}: 135 fiat currencies with real-time
    exchange rates.
    \item \textbf{Embeddable}: Inline, modal, or redirect payment flows.
    \item \textbf{SDK support}: JavaScript, React, iOS, Android.
\end{itemize}

The crypto payment extension maintains these properties. A merchant
enabling crypto payments adds a single configuration flag:

\begin{verbatim}
Hanzo.checkout({
  merchantId: 'merch_abc123',
  amount: 99.99,
  currency: 'USD',
  crypto: {
    enabled: true,
    accept: ['BTC', 'ETH', 'USDC', 'USDT'],
    chain: ['lux', 'ethereum', 'bitcoin', 'solana'],
    settlement: 'USDC'
  }
});
\end{verbatim}

\section{Crypto Payment Architecture}
\label{sec:architecture}
% ============================================================================

\subsection{Payment Flow}

\begin{enumerate}[leftmargin=1.4em]
    \item \textbf{Initiation}: Customer selects crypto payment at
    checkout. SDK displays available currencies and chains.
    \item \textbf{Rate lock}: Hanzo locks the exchange rate for 15
    minutes. Rate sourced from aggregated exchange feeds (Binance,
    Coinbase, Kraken) with TWAP smoothing.
    \item \textbf{Address generation}: MPC-derived payment address
    generated for this transaction (unique per payment).
    \item \textbf{Payment}: Customer sends cryptocurrency to the
    generated address.
    \item \textbf{Confirmation}: Chain-specific confirmation monitoring
    (see Section~\ref{sec:multichain}).
    \item \textbf{Settlement}: Received crypto is converted to the
    merchant's settlement currency (default USDC) and transferred to
    the merchant's MPC-managed settlement wallet.
    \item \textbf{Fiat off-ramp}: If configured, USDC is converted to
    fiat and wire-transferred to the merchant's bank account within 24
    hours.
\end{enumerate}

\subsection{MPC Wallet Management}

Each merchant's payment infrastructure is backed by an MPC key group
using CGGMP21 threshold ECDSA~\cite{cggmp21}:

\begin{itemize}[leftmargin=1.4em]
    \item \textbf{2-of-3 signing}: Hanzo Operations + Merchant Co-Signer
    + Disaster Recovery.
    \item \textbf{Single DKG ceremony}: One key generation produces
    signing capability for all supported chains (secp256k1 for EVM and
    Bitcoin, ed25519 for Solana via separate ceremony).
    \item \textbf{Payment addresses}: Derived from the MPC public key
    using chain-specific address encoding (Ethereum \texttt{keccak256},
    Bitcoin \texttt{bech32}, Solana \texttt{base58}).
    \item \textbf{Unique addresses}: Each payment gets a unique derived
    address via BIP-32 HD derivation from the MPC master key.
\end{itemize}

\section{Instant Settlement on Lux}
\label{sec:lux}
% ============================================================================

For payments on Lux C-Chain, settlement is near-instant:

\begin{itemize}[leftmargin=1.4em]
    \item \textbf{Finality}: Lux consensus provides sub-second deterministic
    finality. Once a transaction is confirmed, it cannot be reverted.
    \item \textbf{Gas cost}: $<$\$0.01 per transaction.
    \item \textbf{USDC native}: USDC is deployed natively on Lux C-Chain,
    avoiding bridge risk.
    \item \textbf{Merchant notification}: Webhook fired to merchant
    within 2 seconds of payment.
\end{itemize}

For merchants prioritizing settlement speed, the checkout SDK
preferentially suggests Lux C-Chain as the payment network while
maintaining support for all configured chains.

% ============================================================================
\section{Fiat Off-Ramps}
\label{sec:offramp}
% ============================================================================

Merchants requiring fiat settlement receive automated off-ramp
processing:

\begin{enumerate}[leftmargin=1.4em]
    \item Crypto payments accumulate in the merchant's MPC settlement
    wallet as USDC.
    \item At configurable intervals (hourly, daily, or threshold-based),
    the settlement service initiates an off-ramp.
    \item USDC is redeemed through Circle's institutional redemption
    channel or converted via a regulated exchange partner.
    \item USD wire transfer sent to the merchant's verified bank account.
    \item End-to-end: crypto payment to bank deposit within 24 hours.
\end{enumerate}

For merchants comfortable holding USDC, the off-ramp step is skipped
and settlement completes on-chain.

\section{Merchant APIs}
\label{sec:merchantapi}
% ============================================================================

The crypto payment extension integrates with the Hanzo Commerce API:

\begin{description}[leftmargin=1.4em]
    \item[\texttt{POST /v1/payments}] Create a payment intent with crypto
    options. Returns payment addresses for each configured chain.
    \item[\texttt{GET /v1/payments/:id}] Check payment status.
    Confirmation count, settlement status, exchange rate used.
    \item[\texttt{POST /v1/payments/:id/refund}] Initiate a refund.
    MPC-signed transaction sends crypto back to the customer's address.
    \item[\texttt{GET /v1/settlements}] List settlements with timestamps,
    amounts, and bank transfer references.
    \item[\texttt{POST /v1/webhooks}] Configure webhooks for payment
    events: \texttt{payment.detected}, \texttt{payment.confirmed},
    \texttt{payment.settled}, \texttt{payment.refunded}.
\end{description}

All endpoints require Hanzo IAM authentication via OAuth 2.0 bearer
tokens. Merchant API keys are scoped to specific operations (read-only,
payment, settlement).

\section{Multi-Chain Support}
\label{sec:multichain}
% ============================================================================

\subsection{EVM Chains (Lux, Ethereum)}

\begin{itemize}[leftmargin=1.4em]
    \item Address derivation: \texttt{keccak256(pubkey)[12:]} from MPC
    public key.
    \item Token support: native ETH/LUX + ERC-20 (USDC, USDT).
    \item Confirmation policy: 1 block on Lux (final), 12 blocks on
    Ethereum ($\sim$3 minutes).
    \item Gas estimation: real-time, paid from merchant's gas wallet
    (auto-funded from settlement).
\end{itemize}

\subsection{Bitcoin}

\begin{itemize}[leftmargin=1.4em]
    \item Address derivation: \texttt{bech32} (native SegWit P2WPKH) from
    compressed MPC public key.
    \item Confirmation policy: 1 confirmation for amounts $<$\$1,000, 3
    confirmations for $<$\$10,000, 6 confirmations for $\geq$\$10,000.
    \item Fee estimation: mempool-based fee rate selection targeting next-block
    inclusion.
    \item UTXO management: automated consolidation during low-fee periods.
\end{itemize}

\subsection{Solana}

\begin{itemize}[leftmargin=1.4em]
    \item Separate ed25519 MPC ceremony (CGGMP21 adapted for ed25519).
    \item SPL token support: USDC, USDT on Solana.
    \item Confirmation: single slot ($\sim$400ms).
    \item Transaction priority: Jito tips for guaranteed inclusion during
    congestion.
\end{itemize}

\section{Payment Channels}
\label{sec:channels}
% ============================================================================

For high-frequency merchants (e.g., SaaS subscriptions, streaming
payments), Hanzo supports payment channels on Lux C-Chain:

\begin{definition}[Payment Channel]
A payment channel is a bilateral off-chain state channel between a
customer and the merchant's MPC wallet. The customer deposits funds into
an on-chain escrow contract, then sends signed state updates (payment
increments) off-chain. The merchant can close the channel at any time,
settling the final state on-chain.
\end{definition}

Payment channel properties:
\begin{itemize}[leftmargin=1.4em]
    \item \textbf{Instant}: Off-chain updates require no block
    confirmation.
    \item \textbf{Micro-payments}: No per-transaction gas cost until
    channel close.
    \item \textbf{Dispute resolution}: On-chain timeout-based dispute
    mechanism with 24-hour challenge period.
    \item \textbf{Batch settlement}: Multiple micro-payments settle in
    a single on-chain transaction.
\end{itemize}

\section{Integration with Hanzo Commerce}
\label{sec:integration}
% ============================================================================

The crypto payment extension is a module within the Hanzo Commerce stack:

\begin{itemize}[leftmargin=1.4em]
    \item \textbf{Hanzo IAM}: Merchant authentication and API key
    management. OAuth 2.0 with org-scoped permissions.
    \item \textbf{Hanzo KMS}: Encryption of MPC key material at rest.
    All merchant key shares encrypted with KMS-managed keys.
    \item \textbf{Hanzo Base}: Merchant configuration, payment records,
    and settlement history stored in Base (SQLite for local dev,
    PostgreSQL for production multi-instance).
    \item \textbf{Hanzo Gateway}: Rate limiting, request routing, and
    TLS termination for all merchant API traffic.
    \item \textbf{Hanzo Webhooks}: Event delivery for payment lifecycle
    events with guaranteed at-least-once delivery and HMAC signature
    verification.
\end{itemize}

\section{Conclusion}
\label{sec:conclusion}
% ============================================================================

Hanzo Crypto Payment Rails extends the proven checkout.js commerce
platform with multi-chain cryptocurrency acceptance. MPC custody
eliminates the key management burden for merchants, while the
integration with Hanzo Commerce API provides a unified interface for
fiat and crypto payments. Lux Network's sub-second finality makes it
the preferred settlement layer, with Bitcoin, Ethereum, and Solana
supported for customer flexibility. Production deployments demonstrate
that crypto payments can match the merchant experience of traditional
card payments: drop-in SDK integration, instant notification, and
next-day fiat settlement.
